install self-hosted directus

June 6, 2023

install self-hosted directus

ubuntu 22.04에 directus를 docker-compose를 이용하여 설치 운영한다. directus에서 제공하는 설치 가이드를 참고하여 진행한다.

directus는 node.js와 vue.js 3로 순수 javascript로 만든 데이터 플랫폼으로 백엔드 api 서버로 사용하고자 한다.

docker와 docker-compose를 설치하고 별도로 설치된 postgresql 서버에 접속하여 운영되도록 한다. nginx로 proxy 설정하여 다른 웹서비스와 하위 경로로 구분되도록 한다.

docker 설치

docker를 설치하는 쉡스크립트(get-docker.sh)를 먼저 다운로드 후 해당 쉘스크립트를 실행하여 설치한다.

$ chmod 755 get-docker.sh

docker-compose 설치

sudo apt install docker-compose

directus docker 설정

directus 폴더를 만들고 docker-compose.yml로 설정한다. 아래 설정은 sqlite를 DB로 이용한다. 정상적으로 실행되는지 확인하고 나중에 실제로 사용할 postgresql 서버로 변경한다.

docker-compose.yml
version: '3'
services:
  directus:
    image: directus/directus:latest
    ports:
      - 8055:8055
    volumes:
      - ./uploads:/directus/uploads
      - ./database:/directus/database
    environment:
      KEY: '<유니크한 키입력>'
      SECRET: '<KEY에 대한 암호입력>'

      DB_CLIENT: 'sqlite3'
      DB_FILENAME: './database/data.db'

      ADMIN_EMAIL: 'appaga@appaga.kr'
      ADMIN_PASSWORD: '<암호입력>'

docker-compse로 실행(up)하고 종료(down)하면 된다. 실행 시에 -d로 백그라운드로 실행시킬 수 있다.

$ pwd
/home/ubuntu/directus

$ docker-compose up -d
Creating network "directus_default" with the default driver
Creating directus_directus_1 ... done

$ docker-compose down
Stopping directus_directus_1 ... done
Removing directus_directus_1 ... done
Removing network directus_default

directus용 postgresql DB 생성

docker로 구동된 서버에서 localhost에 접속하기 위해서는 DB_HOST를 127.0.0.1이 아닌 host.docker.internal로 설정해야 한다. docker로 구동된 서버에서 localhost는 자기 자신에 해당하지 ubuntu 서버를 의미하지 않기 때문이다.

extra_hosts에 설정을 추가해준다.

directus:
    extra_hosts:
      - "host.docker.internal:host-gateway"

postgresql 설정을 추가해주고 sqlite용 설정은 삭제한다.

DB_CLIENT: 'postgres'
DB_HOST: 'host.docker.internal'
DB_PORT: '5432'
DB_DATABASE: 'appagakr_db'
DB_USER: 'appagakr_db'
DB_PASSWORD: '<암호입력>'

암호(DB_PASSWORD)에 처음에는 특수문자 !@#$ 들을 포함하였더니 인증 실패가 발생하였다. @만 포함하고 일반문자만 이용하여 암호를 만들었더니 정상 접속이 되었다. 아무래도 스크립트를 깨드리는 특수문자를 포함하여 암호를 만들면 안되겠다.

directus config options

directus 설정 가이드를 참고하여 몇 가지 일반 설정을 변경한다.

일반 설정 변경

PUBLIC_URL: https://appaga.kr/directus으로 사용하는 루트 url을 설정한다.

ROOT_REDIRECT: false로 설정하여 루트 url(PUBLIC_URL)을 직접 접속시에 자동으로 관리자 페이지로 이동하는 기능을 끈다.

최종 전체 설정

ubuntu 서버에 별도로 설치된 postgresql DB를 사용하도록 설정된 전체 설정 파일이다.

docker-compose.yml
version: '3'
services:
  directus:
    image: directus/directus:latest
    ports:
      - 8055:8055
    volumes:
      - ./uploads:/directus/uploads
    extra_hosts:
      - host.docker.internal:host-gateway
    environment:
      KEY: '122d861b-e4a1-1996-9a13-812330ec40b1'
      SECRET: '2336487b--512c2-b515-a9122c45e263'

            PUBLIC_URL: 'https://appaga.kr/directus'
      ROOT_REDIRECT: 'false'
      LOG_LEVEL: 'info'

      ADMIN_EMAIL: 'appaga@appaga.kr'
      ADMIN_PASSWORD: '<암호입력>'

      DB_CLIENT: 'pg'
      DB_HOST: 'host.docker.internal'
      DB_PORT: '5432'
      DB_DATABASE: 'appagakr_db'
      DB_USER: 'appagakr_db'
      DB_PASSWORD: '<암호입력>'

Nginx 설정

Nginx의 proxy를 이용하여 directus 실행 url이 루트(/)가 아닌 하위 경로(/main)로 실행되도록 한다.을 rewrite 룰을 이용해야 한다. directus는 context-path를 설정하는 옵션은 없어 보인다.

server {
        listen 443 ssl http2;
        listen [::]:443 ssl http2;

        server_name appaga.kr www.appaga.kr;

        root /var/www/html;

        index index.html index.htm index.nginx-debian.html;


        ssl on;
        ssl_certificate /etc/nginx/ssl/dodevops.crt;
        ssl_certificate_key /etc/nginx/ssl/dodevops.csr;

        # deny access to .htaccess files, if Apache's document root
        # concurs with nginx's one
        location ~ /\.ht {
               deny all;
        }

        location / {
                # First attempt to serve request as file, then
                # as directory, then fall back to displaying a 404.
               try_files $uri $uri/ =404;
        }


        # directus conf : start
        location /main {
                rewrite ^/main/(.*)$ /$1 break;
                proxy_pass http://127.0.0.1:8055;
                proxy_set_header Host $host;
                proxy_set_header X-Real-IP $remote_addr;
                proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
                proxy_set_header REMOTE-HOST $remote_addr;
                add_header X-Cache $upstream_cache_status;

                proxy_connect_timeout 30s;
                proxy_read_timeout 86400s;
                proxy_send_timeout 30s;
                proxy_http_version 1.1;
                proxy_set_header Upgrade $http_upgrade;
                proxy_set_header Connection "upgrade";
        }
        # directus conf : end

        client_max_body_size 50m;
        access_log  /var/log/nginx/directus.log;
        error_log  /var/log/nginx/directus_error.log;
}

systemctl 명령으로 재실행한다.

sudo systemctl restart nginx

설정된 관리자 페이지 https://appaga.kr/main/admin로 접속하여 ADMIN_EMAIL로 로그인이 가능하다.

다음으로는 https://docs.directus.io/guides/headless-cms/build-static-website/nuxt-3.html 문서를 참조하여서 nuxt로 프론트엔드를 구현하는 방법을 진행하고자 한다.
