아마존 EC2 Ubuntu 초기 설정

July 15, 2023

아마존 EC2 Ubuntu 초기 설정

ubuntu 22.04 서버 설치 후 기본 설정한 내용을 정리한다. 타임존, 호스트명, swap 메모리, nginx를 설치한다.

시스템 업데이트

신규 패키지 설치전에는 먼저 시스템을 최신으로 업데이트하고 시작한다.

sudo apt update
sudo apt -y upgrade

timezone 설정

타임존을 Asia/Seoul로 변경한다.

date 명령으로 시간을 확인해보면 UTC로 타임존이 설정되어 있다. timedatectl 명령으로 Asia/Seoul로 변경한다.

appaga@appagakr:~$ date
Sat Apr 30 01:04:52 PM UTC 2022
appaga@appagakr:~$ sudo timedatectl set-timezone Asia/Seoul
[sudo] password for appaga:
appaga@appagakr:~$ date
Sat Apr 30 10:06:01 PM KST 2022

hostname 설정

hostname을 이용하여 서버명을 설정한다.

$ sudo hostname appaga.kr
$ hostname
appaga.kr

swap 파일 설정

물리 메모리가 1GB이고 적당한 크기인 2GB의 스왑 파일을 설정하여 사용한다.

스왑이 설정되어 있는지 확인

sudo swapon --show

메모리 용량을 체크하여도 확인 가능하다.

$ free -h
               total        used        free      shared  buff/cache   available
Mem:           966Mi       203Mi        77Mi       0.0Ki       685Mi       610Mi
Swap:             0B          0B          0B

스왑파일 생성하기

물리 메모리 용량 1GB의 두 배인 2GB 용량의 스왑파일을 생성한다.

$ sudo fallocate -l 2G /swapfile
$ ls -lh /swapfile
-rw-r--r-- 1 root root 2.0G Mar 18 11:13 /swapfile

스왑파일 활성 시키기

스왑파일을 스왑공간으로 활성 시키고 root 사용자만 접근이 가능하도록 권한 설정을 한다.

$ sudo chmod 600 /swapfile
$ ls -lh /swapfile
-rw------- 1 root root 2.0G Mar 18 11:13 /swapfile
$ sudo mkswap /swapfile
Setting up swapspace version 1, size = 2 GiB (2147479552 bytes)
no label, UUID=59c04b6e-a1c2-4a7b-af3d-f5afebbcba6b
$ sudo swapon /swapfile
$ sudo swapon --show
NAME      TYPE SIZE USED PRIO
/swapfile file   2G   0B   -2
$ free -h
               total        used        free      shared  buff/cache   available
Mem:           966Mi       202Mi        77Mi       0.0Ki       686Mi       612Mi
Swap:          2.0Gi          0B       2.0Gi

스왑공간을 기본으로 사용

서버가 재기동 되어도 스왑공간을 계속 사용되도록 설정한다. /etc/fstab 파일을 백업 후에 스왑공간 설정을 추가한다.

$ sudo cp /etc/fstab /etc/fstab.bak
$ echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
/swapfile none swap sw 0 0
$ cat /etc/fstab
LABEL=cloudimg-rootfs   /        ext4   discard,errors=remount-ro       0 1
LABEL=UEFI      /boot/efi       vfat    umask=0077      0 1
/swapfile none swap sw 0 0

스왑 빈도율 설정

물리 램과 스왑공간 간에 데이터 스왑하는 빈도를 조정한다. 0~100 퍼센트 값을 설정 가능하고 60은 데스크탑용에 적당하고 서버는 0에 가까와야 좋다. 10으로 설정한다.

$ cat /proc/sys/vm/swappiness
60
$ sudo sysctl vm.swappiness=10
vm.swappiness = 10

서버 재기동 후에도 적용되도록 설정한다. /etc/sysctl.conf 파일 마지막 줄에 vm.swappiness=10 설정을 추가한다.

/etc/sysctl.conf
$ sudo vi /etc/sysctl.conf

vm.swappiness=10

서버 재기동 후에 확인

서버 재기동 후에도 스왑 설정이 유지되는지 확인한다.

$ sudo reboot now

$ sudo swapon --show
NAME      TYPE SIZE USED PRIO
/swapfile file   2G   0B   -2
$ free -h
               total        used        free      shared  buff/cache   available
Mem:           966Mi       182Mi       436Mi       0.0Ki       346Mi       634Mi
Swap:          2.0Gi          0B       2.0Gi
$ cat /proc/sys/vm/swappiness
10

nginx 설치

nginx 설치 후 서비스 자동 실행으로 등록한다.

sudo apt install nginx
sudo systemctrl enable nginx

CSR (인증서 서명 요청) 생성

SSL을 요청할 수 있으려면 먼저 자신의 서버로부터 인증서 서명 요청(CSR)을 생성해야 한다.

openssl 명령으로 생성하며 암호는 선택 입력 사항이지만 보안 강화를 위해서 입력하는 것을 추천한다.

$ openssl req -new -newkey rsa:2048 -nodes -keyout appaga.kr.csr -out appaga.kr.csr

ssl 파일 설치

nginx 설정 폴더인 /etc/nginx 폴더 하위에 SSL 인증 파일을 저장할 ssl 폴더를 생성하고 파일들을 복사한다.

mkdir /etc/nginx/ssl
sudo cp appaga.kr.crt /etc/nginx/ssl/
sudo cp appaga.kr.csr /etc/nginx/ssl/

ssl 설정 적용

/etc/nginx/sites-available/ 폴더에 ssl 설정 파일 appaga.kr-443port을 생성하고 아래 내용으로 설정한다.

server {
        listen 443;
        listen [::]:443;

        root /var/www/html;

        index index.html index.htm index.nginx-debian.html;

        server_name appaga.kr www.appaga.kr;

        ssl on;
        ssl_certificate /etc/nginx/ssl/appaga.kr.crt;
        ssl_certificate_key /etc/nginx/ssl/appaga.kr.csr;

        location / {
                # First attempt to serve request as file, then
                # as directory, then fall back to displaying a 404.
                try_files $uri $uri/ =404;
        }

        # deny access to .htaccess files, if Apache's document root
        # concurs with nginx's one

        location ~ /\.ht {
               deny all;
        }

}

/etc/nginx/sites-enabled 폴더에 링크를 걸어주면 설정이 적용된다.

cd /etc/nginx/sites-enabled
sudo ln -s ../sites-available/appaga.kr-443port ./
sudo systemctl reload nginx

80 포트를 443 포트로 리다이렉트 하기

80 포트로 들어오는 모든 요청을 443 포트로 리다이렉트하여 443 포트만 사용하도록 한다.

00appaga.kr-80port 파일을 만들어 설정한다.

# redirecting from HTTP to HTTPS
#
server {
        listen 80;
        listen [::]:80;

        root /var/www/html;

        index index.html index.htm index.nginx-debian.html;

        server_name appaga.kr www.appaga.kr;

        return 301 https://$host$request_uri; # Redirect
}

/etc/nginx/sites-enabled 폴더에 링크를 걸어주면 설정이 적용된다.

cd /etc/nginx/sites-enabled
sudo ln -s ../sites-available/appaga.kr-80port ./
sudo systemctl reload nginx
